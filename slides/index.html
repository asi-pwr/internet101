TCP & UDP

TCP

A client-server protocol

Server awaits connection at a certain point. Once a client initiates the connection the transmission starts. In contrast to UDP, TCP makes sure that ALL packets will arrive in order and with no duplicates. For every packet sent the target device sends an acknowledgement. In case of missing packets a retransmission request can be issued

UDP

A connectionless protocol

Just send the packet along with a checksum. Gives no guarantees the packets will reach the target
